About toon-memory

Persistent memory for AI coding agents — remember decisions, patterns, and bugs between sessions.

- What is toon-memory? - Blog Post - Features - Quick Start - Supported Agents - MCP Tools - Tips & Best Practices - CLI Commands - Configuration - How It Works - Why TOON? - Troubleshooting - FAQ - Development - Contributing - License

Ever had that feeling where your AI agent forgets everything from yesterday's session? You explain the same architecture decision for the third time, and it still suggests the approach you already rejected?

toon-memory fixes this. It gives your AI agent a persistent memory that survives restarts, so it actually learns from your project over time.

Scenario What toon-memory does -------------------------------- Design debates "We chose Redis over Memcached because of pub/sub support" Framework choices "This project uses Zod for validation, not Joi" Bug fixes "Redis pool exhaustion — fix was maxconnections=20" Architecture notes "Broker service uses RESP protocol, not HTTP" Onboarding "The deploy script lives in scripts/deploy.sh" Team context "PR 142 reverted the caching change — don't re-add it"
