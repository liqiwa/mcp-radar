About ruyi-mcp

ruyi-mcp 是面向 ruyiPage 的社区 MCP Server，将 Firefox / WebDriver BiDi 浏览器自动化、运行时观察、指纹分析、Trace、网络拦截和人类行为模拟能力提供给 Claude Code、Codex、Cursor 等 MCP 客户端。

- 提供 57 个 MCP 工具，覆盖页面生命周期、脚本与运行时分析、网络抓取、Cookie、DOM、Frame、请求/响应拦截、WebSocket、浏览器指纹、人类行为模拟、Session 导出和 Trace。 - ruyihumandrag 提供原子的拟人拖拽动作链；ruyisetfingerprint.windowSize 可同步 window、viewport 与 screen 指标。 - Node.js MCP Server 通过常驻 Python JSON-RPC Bridge 调用 ruyiPage。 - 仓库跟踪 TypeScript 构建产物；依赖安装完成后，MCP Host 可直接从 build/src/index.js 启动。

- Node.js 20 或更高版本。 - Python 3.10 或更高版本；当前已在 Python 3.13 上验证。 - ruyiPage==1.2.50，以及由 ruyiPage 安装或兼容的 Firefox。

ruyi-mcp ruyiPage Node.js Python 验证环境 ----------------------------------------------- v0.1.2 1.2.50 =20 =3.10 GitHub Actions：Node.js 20 + Python 3.13

仓库对 ruyiPage 使用精确版本锁定。升级兼容版本前会重新执行 Bridge contract、TypeScript build 和 57 tools stdio smoke test。

- RUYIMCPPYTHON：Node Bridge 使用的 Python 可执行文件。Windows 默认使用 python，其他平台默认使用 python3。 - RUYIFIREFOXPATH：Firefox 可执行文件路径。未设置时，Bridge 会依次检查 reverseENV 便携目录、Windows RuyiPage 浏览器缓存和 PATH。

- ruyitrace 暴露的是 RuyiPage 内存中的 WebDriver BiDi JSON Trace，不是 Firefox 内核 DOMTrace。 - ruyiPage 1.2.50 的安装器仍使用 151-ruyi runtime；如需验证 credentialed HTTP / SOCKS5 proxy，应单独下载上游 151-proxy release，并通过 RUYIFIREFOXPATH 显式指向解压后的 firefox.exe。 - 在 151-proxy 上同步窗口尺寸时，browsingContext.setViewport 可能等待 3 秒后由 ruyiPage 自动切换到 JS viewport fallback；这是兼容路径，不影响后续浏览器复用。 - 本仓库不分发 Firefox 二进制、浏览器 Profile 或 DOMTrace 内核。
